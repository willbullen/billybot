Settings & System Documentation

Overview

The Settings page provides system-wide management including Docker container control, health monitoring, launch configuration reference, environment variables, and deployment controls.

Health data refreshes automatically every 15 seconds via /api/system/health/.

Containers

Live status of all Docker containers in the BillyBot stack. Each container can be individually restarted.

billybot-ros2
ROS 2 Humble container. Runs all robot nodes: motor drivers, audio pipeline, voice agents, SLAM, Nav2, behavior manager. Uses network_mode: host for DDS multicast.
billybot-nanobot
Nanobot AI gateway. Provides multi-model LLM access with tools, memory, and scheduled jobs. Connects to ROS 2 via the ros2 tool.
billybot-dashboard
Django + Channels + Daphne. Serves the web dashboard, REST APIs, and WebSocket connections. Communicates with ROS 2 container via docker exec.

Restart Warning: Restarting the ROS 2 container will stop all robot nodes, interrupting motor control, navigation, and voice processing. The robot will stop moving. Nodes auto-restart when the container comes back up.

System Health

Aggregated health status across all subsystems. The overall badge shows:

HEALTHY
All containers running, ROS 2 nodes active, topics publishing.
DEGRADED
Some containers or nodes not responding. Partial functionality available.
UNHEALTHY
Critical failure. Core containers or nodes are down.
Dashboard Uptime
Time since the Django process started. Format: days/hours/minutes/seconds.
ROS 2 Nodes
Count of active nodes from ros2 node list. Listed individually below.
ROS 2 Topics
Total active topic count from ros2 topic list.
Containers Running
Running containers vs total containers (e.g., 3/3).

Launch Configurations

Reference table of available ROS 2 launch files in the by_your_command package:

oai_realtime.launch.py
OpenAI Realtime API for voice-only conversation with function calling.
oai_dual_agent.launch.py
OpenAI dual agent: voice conversation + parallel command extraction pipeline.
gemini_live.launch.py
Gemini Live API with optional video input from the camera.
gemini_dual_agent.launch.py
Gemini dual agent: full multimodal conversation + command extraction.
gemini_vision.launch.py
Gemini vision-focused: periodic camera frame analysis and scene description.
hardware.launch.py
Hardware layer only: URDF, DDSM210 motors, ST3215 servos. No AI.
camera.launch.py
RealSense D435i driver. Use simulate:=true for development without camera.
navigation.launch.py
Full nav stack: hardware + camera + slam_toolbox + Nav2. Required for autonomous navigation.

Launch files are started inside the ROS 2 container:
ros2 launch by_your_command <file>.launch.py

Pass parameters with key:=value syntax, e.g., namespace:=grunt2 simulate:=true

System Info

ROS 2 Version
Humble Hawksbill (LTS). Ubuntu 22.04 base image.
ROS Domain ID
DDS domain for node discovery. All robots must share the same ID (default: 0). Set via ROS_DOMAIN_ID environment variable.
DDS Middleware
CycloneDDS (rmw_cyclonedds_cpp). Selected for reliable multicast discovery and shared-memory transport.
Bridge WebSocket
WebSocket endpoint for rosbridge_server (if enabled). Used by external tools like foxglove.
Dashboard Stack
Django 4.x + Channels (ASGI) + Daphne (WebSocket server). Alpine.js for reactive UI, Tailwind CSS for styling.

Environment Variables

ROS_DOMAIN_ID
DDS domain for multi-robot isolation. Default: 0.
RMW_IMPLEMENTATION
DDS middleware. Always rmw_cyclonedds_cpp.
ROS2_BRIDGE_WS
WebSocket URL for rosbridge_server connections.
OPENAI_API_KEY
API key for OpenAI models (GPT-4o, Realtime API). Masked in UI.
GOOGLE_API_KEY
API key for Google Gemini models. Masked in UI.

Deployment

Build & Deploy
Triggers a full build and redeploy of all containers. Runs scripts/deploy.sh on the host.
Pull & Deploy
Pulls latest images from registry and redeploys. Use when pre-built images are available.
Rollback
Reverts to the previous deployment. Use if a new deploy introduces issues.

Note: Deployment commands are placeholders that show the host-side command to run. Actual deployment must be triggered from the host machine, not from within the container.

API Endpoints

GET /api/docker/containers/
Returns list of containers with status, name, and running state.
POST /api/docker/restart/
Restarts a container. Body: { container: "name" }.
GET /api/system/health/
Returns aggregated health: overall status, uptime, node count, topic count, container states.
