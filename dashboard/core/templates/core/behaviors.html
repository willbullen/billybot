Autonomous Behaviors Documentation

Overview

The Behaviors page provides command and monitoring for the behavior_manager_node, a finite state machine (FSM) that orchestrates high-level autonomous robot behaviors. The node ticks at 5 Hz and publishes status at 1 Hz to /behavior_status.

Commands are sent via the BehaviorConsumer WebSocket (ws/behaviors/) which publishes to /grunt1/behavior_command.

States

The behavior manager operates as a finite state machine with five states:

IDLE
Default state. Robot is stationary, no active behavior. All behaviors transition here on stop command.
PATROL
Cycles through waypoints using Nav2 goal publishing. Loops continuously. 30-second timeout per waypoint — auto-skips if stuck.
SEARCH
Expanding spiral pattern from current position. Covers configurable radius in rings of 4 points each. Returns to origin when complete.
FOLLOW
Tracks a detected target (from /detected_target topic) while maintaining safe distance. Rotates to face target when close, navigates toward it when far.
GUARD
Stations at current position and periodically scans N/E/S/W by rotating the camera arm. Auto-returns to guard position if displaced >0.5m.

Patrol Behavior (Detail)

Default waypoints form a 2m × 2m square pattern. Each waypoint specifies (x, y, yaw) in the map frame.

Loop Mode
When enabled (default), patrol restarts from waypoint 1 after reaching the last waypoint.
Timeout
30 seconds per waypoint. If the robot cannot reach a waypoint (obstacle, nav failure), it skips to the next.
Goal Tolerance
0.3m — the robot considers a waypoint reached when within this distance.
Custom Waypoints
Add custom waypoints via the UI panel. Enter X, Y, Yaw values and click Add.

Search Behavior (Detail)

Pattern
Expanding spiral: 4 points per ring at 90° intervals, increasing radius each ring.
Search Radius
Default 3.0m from origin. Configurable via search_radius parameter.
Step Size
Default 1.0m between rings. Configurable via search_step parameter.
Camera
Arm set to lookup position for maximum field of view during search.

Follow Behavior (Detail)

Target Source
Subscribes to /detected_target (String JSON with label, x, y fields). Vision pipeline publishes detected object positions here.
Follow Distance
Default 1.5m. Robot navigates to maintain this distance from target.
Close Range
Within follow_distance: robot rotates to face target (proportional yaw control).
Far Range
Beyond 1.5× follow_distance: robot publishes nav goal toward target.

Guard Behavior (Detail)

Scan Cycle
Every 10 seconds, rotates camera to next cardinal direction: North → East → South → West.
Position Hold
If odometry shows >0.5m drift from guard position, auto-navigates back.
Camera Control
Uses pan@forward, pan@right, pan@back, pan@left arm presets for directional scanning.

Voice Commands

Behaviors can be triggered via voice through the AI agent pipeline:

patrol
Start patrol with default waypoints.
search
Start spiral search from current position.
follow@person
Start following a “person” target. Any label accepted.
guard
Start guarding current position.
stop
Emergency stop — halts any active behavior and returns to IDLE.

Configuration

File: ros/config/behaviors.yaml

Launch: ros2 launch by_your_command behaviors.launch.py simulate:=true

Includes full navigation stack + behavior manager node.
