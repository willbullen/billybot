Telemetry Documentation

Overview

The Telemetry page provides real-time data visualization of robot subsystem performance. Data streams via the TelemetryConsumer WebSocket (ws/telemetry/) at approximately 1 Hz.

Charts are rendered with Chart.js and show rolling 60-sample windows.

Data Source

The TelemetryConsumer polls the /motor_feedback topic (std_msgs/String containing JSON) via docker exec inside billybot-ros2. The DDSM driver node publishes this data from motor encoder feedback.

If hardware is not connected, the DDSM driver runs in simulation mode and publishes synthetic feedback data (zero speeds, simulated temperatures).

Motor Speed Chart

Displays left and right motor speeds in RPM as reported by the DDSM210 hub motor encoders.

Left Speed (blue)
Left-side wheel pair RPM. Positive = forward. Range: -2100 to 2100 RPM.
Right Speed (cyan)
Right-side wheel pair RPM. Positive = forward. Range: -2100 to 2100 RPM.
Correlation
Equal values = straight line. Opposite values = spin in place. Differential = turning arc.

Temperature Chart

Motor controller temperatures from the DDSM Driver HAT. Thermal monitoring prevents motor damage from overheating.

Left Temp (amber)
Left motor controller temperature in °C.
Right Temp (red)
Right motor controller temperature in °C.
Warning Threshold
Temperatures above 70°C indicate sustained high-current operation. Above 85°C triggers thermal protection.

Audio Energy & VAD

Voice Activity Detection (VAD) state from the Silero VAD node. Shows audio energy levels and speech detection status.

Energy Level
RMS audio energy from the GStreamer capture pipeline. Higher values indicate louder audio.
VAD State
Silero neural network speech detection: ACTIVE (speech detected) or SILENT (no speech).
Voice Pipeline
GStreamer → Silero VAD → WebSocket Bridge → AI Agent (OpenAI/Gemini).

Hardware Reference

Motors
DDSM210 hub motors. 4WD skid-steer. JSON over UART @ 115200 baud via DDSM Driver HAT.
Encoder
4096 counts/revolution. Provides speed feedback and odometry data.
Protocol
JSON commands: {"T":1, "L":<rpm*10>, "R":<rpm*10>}
