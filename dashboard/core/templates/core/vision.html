Vision & Navigation Documentation

Overview

The Vision & Navigation page integrates live camera streaming, SLAM mapping, and autonomous navigation into a unified interface. It connects to two WebSocket streams: ws/camera/ for video frames and ws/map/ for the SLAM map and robot pose.

Camera Feed

Streams compressed JPEG frames from the RealSense D435i camera (or simulation fallback) via the CameraConsumer WebSocket.

Source Topic
/camera/color/compressed (sensor_msgs/CompressedImage)
FPS Control
Adjustable 1-15 fps via dropdown. Higher FPS = more bandwidth. 5 fps recommended for remote operation.
Frame Grab
Each frame is grabbed via an inline Python rclpy subscriber inside billybot-ros2, base64-encoded, and sent over WebSocket.
Simulation
When no camera hardware is present, the camera driver generates a HUD test pattern with crosshair, grid, depth bar, and frame counter.

Navigation Map

Displays the SLAM-generated occupancy grid map with robot position overlay. The MapConsumer polls /map (nav_msgs/OccupancyGrid) and /odom at 1 Hz.

Map Colors
White = free space. Black = obstacles. Gray = unknown/unexplored.
Robot Arrow
Cyan triangle showing current position and heading from odometry.
Click-to-Navigate
Click anywhere on the map to set a navigation goal. The click position is converted to world coordinates using map metadata (resolution, origin).
Goal Marker
Pulsing cyan circle shows the pending navigation goal location.

SLAM & Map Management

SLAM Toolbox runs in async_slam_toolbox_node mode for real-time mapping.

Save Map
Calls /slam_toolbox/save_map service. Saves .pgm + .yaml to /tmp/ in the ROS container.
Load Map
Calls /slam_toolbox/deserialize_map to load a previously saved map for localization.
Map List
The Maps panel shows all .pgm/.yaml files in /tmp/ that can be loaded.

Navigation Controls

Manual navigation goal input using X, Y, and Yaw fields. Goals are published as geometry_msgs/PoseStamped to /goal_pose.

X / Y
World coordinates in meters relative to the map frame origin.
Yaw
Goal orientation in radians. 0 = facing positive X. π/2 = facing positive Y.
Cancel
Cancels the active navigation goal via /navigate_to_pose/_action/cancel_goal.

Navigation Stack

Launch: ros2 launch by_your_command navigation.launch.py simulate:=true

Includes: hardware drivers + camera + depthimage_to_laserscan + SLAM Toolbox + Nav2 (DWB controller, NavFn planner, BT navigator, lifecycle manager).

Voice-to-Nav Commands

The command_processor node routes voice commands to navigation goals:

goto@kitchen
Navigate to a named location. Available: home, kitchen, bedroom, living, door, garage, office, charging.
goto@1.5,2.0
Navigate to specific coordinates (x, y). Optional yaw: goto@1.5,2.0,1.57
patrol
Cycle through predefined patrol waypoints continuously.
